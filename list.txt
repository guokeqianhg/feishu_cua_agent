一、当前项目目标

CUA-Lark 后端目标是实现一个飞书桌面 GUI 智能测试 Agent。

它不调用飞书开放平台 API 完成任务，而是像真实用户一样：
1. 截图观察当前桌面。
2. 理解自然语言任务。
3. 规划多步操作。
4. 用鼠标、键盘真实操作飞书桌面端和浏览器文档编辑器。
5. 每一步截图验证。
6. 生成可回放的 summary.md、summary.json、steps.jsonl 和截图证据。


二、当前真实 E2E 完成情况

1. IM 发送链路：已跑通

能力：
1. 搜索联系人或会话。
2. OCR 校验目标搜索结果行。
3. 打开目标会话。
4. 使用当前输入焦点直接输入消息，避免额外乱点、拖动或点到表情。
5. 安全开关允许后发送消息。
6. 验证消息在目标会话中可见。

关键保护：
1. `CUA_LARK_ALLOW_SEND_MESSAGE=true` 是真实发送总开关。
2. `CUA_LARK_ALLOWED_IM_TARGET` 可限制真实发送目标。
3. OCR 必须确认目标行，避免误点其他搜索结果。
4. 打开会话后直接输入，不再额外点击或长按拖动。

推荐命令：

```powershell
$env:DRY_RUN="false"
$env:CUA_LARK_PLACEHOLDER_SCREENSHOT="false"
$env:CUA_LARK_ALLOW_SEND_MESSAGE="true"
$env:CUA_LARK_ALLOWED_IM_TARGET="吴佳园"
python -B cli.py run --instruction "在IM中搜索吴佳园，发送一条消息：hello" --auto-debug
```

当前流程：
1. 聚焦飞书。
2. 条件式关闭已有浮层或模板预览。
3. 打开云文档。
4. hover “新建”，不盲点。
5. OCR 点击“文档”。
6. OCR 点击“新建空白文档”的加号区域。
7. 新文档在浏览器中打开后保持浏览器前台。
8. 输入标题。
9. Enter 进入正文。
10. 输入正文。
11. 验证标题和正文可见。

推荐命令：

```powershell
$env:DRY_RUN="false"
$env:CUA_LARK_PLACEHOLDER_SCREENSHOT="false"
$env:CUA_LARK_ALLOW_DOC_CREATE="true"
python -B cli.py run-case cases\docs_create_doc.yaml --auto-debug
```

当前流程：
1. 聚焦飞书。
2. 条件式关闭 stale 日程编辑器或确认弹窗。
3. 打开日历。
4. 选中“日历”页签。
5. 点击“创建日程”。
6. 输入标题。
7. 打开开始日期选择器。
8. 通过 OCR 识别日期选择器标题和 7 列日期网格。
9. 根据 `event_date` 计算目标日期格中心，禁止 VLM 猜测兜底。
10. 输入开始时间和结束时间。
11. 保存日程。
12. 等待保存后界面稳定。
13. 本地 OCR 验证日程标题、日期和时间。

八、下一步建议

优先级 1：稳定化与回归
1. 把 IM、Docs、Calendar 三条真实 E2E 各再跑 2-3 次。
2. 汇总失败样本，补齐定位和验证边界。
3. 建立一个只 dry-run / mock 的 CI 风格检查命令。

优先级 2：Accessibility / OCR 融合
1. 引入 Windows UI Automation 获取控件 bbox。
2. 用 OCR 确认文字，用 accessibility 确认控件角色和 bbox。
3. 点击时使用 bbox 中心或安全区域。

优先级 3：更多产品覆盖
1. Base。
2. VC。
3. Mail。

优先级 4：交付材料
1. Demo 视频。
2. 答辩 PPT。
3. system_design.md 补充架构图。
4. evaluation.md 补充真实运行结果表。


九、注意事项

1. 永远不要提交 `.env`。
2. 永远不要提交 API Key、Authorization header、真实 cookies。
3. `runs/` 报告截图可能包含真实桌面内容，默认不要提交到 GitHub。
4. 真实 IM 发送只对测试联系人或测试群执行。
5. Docs / Calendar 创建只使用无害测试标题和测试内容。
6. 如果截图黑屏、窗口不对或 mock 回退，不要强行真实执行。


十、结论

当前项目已经具备可演示的飞书桌面 GUI Agent MVP：
1. 能理解自然语言和 YAML case。
2. 能真实操作飞书。
3. 能覆盖 IM、Docs、Calendar 三个子产品。
4. 能输出可回放报告。
5. 有安全开关和条件式闭环处理。
6. Calendar 日期选择已经从 VLM 猜测改成 OCR 网格定位，避免误点到相邻日期或月份。

下一阶段建议从“能跑通”转向“多次回归稳定”和“更强闭环感知”。
