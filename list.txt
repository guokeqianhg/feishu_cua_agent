一、项目目标

CUA-Lark 后端目标是实现一个飞书桌面 GUI 智能测试 Agent。

它不调用飞书开放平台 API 完成业务动作，而是像真实用户一样：
1. 截图观察当前桌面。
2. 理解自然语言任务。
3. 将任务路由到 IM、Docs、Calendar、VC 等子产品 workflow。
4. 用鼠标、键盘真实操作飞书桌面端和浏览器文档编辑器。
5. 每一步截图验证。
6. 生成可回放的 summary.md、summary.json、steps.jsonl 和截图证据。


二、对照任务规则的完成情况

任务规则要求：
1. 视觉感知：截图识别飞书 UI 元素、状态和布局。
2. 语义理解：理解自然语言测试指令并拆解为操作序列。
3. 自主操作：模拟鼠标、键盘、滚动等完成测试任务。
4. 状态验证：操作后通过视觉或语义判断验证结果。
5. 评估报告：生成操作轨迹、成功率、耗时等报告。
6. 至少覆盖 2 个飞书子产品。

当前完成情况：

7. 进阶能力已部分完成：异常弹窗处理、安全 guard、自愈恢复入口、错误集、OCR/CV/VLM 混合定位。

四、真实运行安全开关

基础真实桌面配置：

```powershell
$env:DRY_RUN="false"
$env:CUA_LARK_PLACEHOLDER_SCREENSHOT="false"
$env:CUA_LARK_MODEL_PROVIDER="auto"
```

IM：

```powershell
$env:CUA_LARK_ALLOW_SEND_MESSAGE="true"
$env:CUA_LARK_ALLOWED_IM_TARGET="测试群"
$env:CUA_LARK_ALLOW_SEND_IMAGE="true"
$env:CUA_LARK_IM_TEST_IMAGE_PATH="assets\im_test_image.png"
$env:CUA_LARK_ALLOW_CREATE_GROUP="true"
$env:CUA_LARK_ALLOWED_GROUP_MEMBER="李新元"
$env:CUA_LARK_ALLOW_EMOJI_REACTION="true"
```

Docs：

```powershell
$env:CUA_LARK_ALLOW_DOC_CREATE="true"
$env:CUA_LARK_ALLOW_DOC_SHARE="true"
$env:CUA_LARK_ALLOWED_DOC_SHARE_RECIPIENT="李新元"
```

Calendar：

```powershell
$env:CUA_LARK_ALLOW_CALENDAR_CREATE="true"
$env:CUA_LARK_ALLOW_CALENDAR_INVITE="true"
$env:CUA_LARK_ALLOW_CALENDAR_MODIFY="true"
```

VC：

```powershell
$env:CUA_LARK_ALLOW_VC_START="true"
$env:CUA_LARK_ALLOW_VC_JOIN="true"
$env:CUA_LARK_ALLOW_VC_DEVICE_TOGGLE="true"
$env:CUA_LARK_VC_MEETING_ID="259427455"
```

安全约束：
1. IM 发送、图片、@、表情等动作前会确认目标会话。
2. 建群成员、Docs 分享对象、IM 发送目标均可通过 allow-list 限制。
3. Calendar 创建、邀请、修改分别用独立 guard 控制。
4. VC 发起、加入、设备切换分别用独立 guard 控制。
5. 截图不健康、模型回退 mock、窗口状态不可信时默认阻断真实执行。

八、报告输出

每次 case 运行会生成：

```text
runs/reports/run_YYYYMMDD_HHMMSS_xxxxxxxx/
|-- summary.json
|-- summary.md
|-- steps.jsonl
|-- screenshots/
`-- artifacts/
```

suite 运行会生成：

```text
runs/reports/suite_YYYYMMDD_HHMMSS_xxxxxxxx/
|-- suite_summary.json
`-- suite_summary.md
```

报告会记录运行模式、真实/模拟状态、截图诊断、每一步 before/after 截图、坐标、输入、耗时、验证结果和失败原因。mock/dry-run 报告会明确提示模拟验证不代表真实飞书操作成功。

十一、注意事项

1. 永远不要提交 `.env`。
2. 永远不要提交 API Key、Authorization header、真实 cookies。
3. `runs/` 报告截图可能包含真实桌面内容，默认不要提交到 GitHub。
4. 真实 IM 发送只对测试联系人或测试群执行。
5. Docs / Calendar 创建只使用无害测试标题和测试内容。
6. Docs 分享只分享给明确测试联系人，例如“李新元”。
7. VC 回归会真实进入会议；每条 start/join case 结束后建议退出会议，避免污染下一条 case。
8. 不要用固定屏幕绝对坐标处理飞书窗口内控件，应继续使用窗口相对定位、OCR、CV 和像素状态。
9. 如果截图黑屏、窗口不对或 mock 回退，不要强行真实执行。
